Manage a Single Published API

This endpoint allows you to manage an API that was published previously using the */apis/* endpoint. You can retrieve information about the API, update it, or delete (retire) it.
PARAMETERS
Parameter Description Type
organizationId The API's organization id. string
apiId The API's unique identifier. string
version The version of the API to get. string
OPERATIONS
GET
Get a Published API
Returns information about a single published API. Required parameters include the organization id, API id, and API version.
PUT
Update a Published API
Updates a single published API with new information, particularly new policy configuration. Use this command when an API's configuration details have changed, such as rate limits or authentication details.
POST
Not Supported
DELETE
Retire an API
Retires an API from the gateway, effectively removing it from the system and making it impossible to invoke.
